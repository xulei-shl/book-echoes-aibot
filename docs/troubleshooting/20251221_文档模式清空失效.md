# 故障复盘报告
- **日期**: 2025-12-21
- **严重级别**: Medium

## 1. 问题现象
- 文档上传模式下，点击 清空 后界面仍显示已上传文档计数 1/5。
- 模式指示器依然停留在“文档”状态，上传文档 按钮持续禁用，无法重新发起上传流程。

## 2. 根本原因 (Root Cause)
- components/aibot/AIBotOverlay.tsx:1217 中的 handleClear 仅重置了聊天消息与深度检索状态，遗漏 esetDocumentAnalysis()、clearUploadedDocuments() 与 setMode(AIBOT_MODES.TEXT)。
- 因此文档分析相关状态（包括模式值、上传列表）没有被清空，UI 仍依据旧状态渲染“文档模式”并禁止上传按钮。

## 3. 解决方案
- 在 handleClear 中补充调用：
  - esetDocumentAnalysis()：恢复文档分析阶段、草稿与日志。
  - clearUploadedDocuments()：清空 uploadedDocuments，让列表恢复空状态。
  - setMode(AIBOT_MODES.TEXT)：强制模式回到简单检索，解除界面禁用。
- 代码位置：components/aibot/AIBotOverlay.tsx。

## 4. 预防措施
- 编写/更新回归测试覆盖“清空”按钮在不同模式下的状态重置（待测试体系完善）。
- 后续修改交互时，统一对 handleClear 做状态清单核对，避免遗漏其他模式状态。
