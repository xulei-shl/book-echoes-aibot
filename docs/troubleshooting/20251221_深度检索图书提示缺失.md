# 故障复盘报告
- **日期**: 2025-12-21
- **严重级别**: Low

## 1. 问题现象
深度检索模式下点击交叉分析报告的“确认检索”后，界面未显示“正在检索相关图书...”提示，用户无法得知检索进度。

## 2. 根本原因 (Root Cause)
组件 `DeepSearchDraftMessage` 使用了 `documentAnalysisPhase` 判断提示展示，而深度检索流程只会更新 `deepSearchPhase`。因为状态来源错误，条件永远为假，导致提示不会出现。

## 3. 解决方案
- 将 `DeepSearchDraftMessage` 中的状态引用改为 `deepSearchPhase`，并基于 `book-search`/`book-selection` 阶段渲染提示。
- 更新相关设计与变更文档以记录正确的状态来源。

## 4. 预防措施
- 在未来包含多模式的界面复用状态时，编写单元或集成测试覆盖关键状态机切换，确保不同模式引用的状态字段明确区分。
- 代码评审时新增检查点：确认 UI 条件渲染所依赖的 store 字段与业务流程中的实际状态机一致。
